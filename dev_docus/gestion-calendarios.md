# GESTIÓN DE CALENDARIOS
 * Solo los admin y root pueden acceder a la gestión de calendarios.
 * Los calendarios abarcan un curso academíco completo: desde el 1 de septiembre de un año hasta el 31 de julio del siguiente año. Ejemplo: curso 2025 - 2026. Desde el 1 de septiembre de 2025 hasta el 31 de julio de 2026.
 * Los admin pueden crear calendarios los cuales pueden definirse como templates o plantillas.
 * Cuando se crea, o amplia un calendario en caso de usarse una plantilla como base  el admin utilizará:
    * Una vista que mostrará los meses y años del curso académico en formato grid. 
    * Todos los días de fin de semana se mostrarán por defecto como no laborables.
    * Podrá seleccionar del grid múltiples días y marcarlos como festivos.
    * Podrá seleccionar del grid múltiples días y marcarlos como periodo de exámenes (en este caso además podrá indicar el tipo mediante un texto: DAM/DAW, junio 1, junio 2, extraordinarios, proyectos...).
 * Un calendario puede estar activo ( y lo verán todos los usuarios) o inactivo. Esta acción la podrá llevar a cabo el admin usando un toogle boton que cambiará de color. (similar a lo que se hace con los usuarios).
 * Los admin pueden crear calendarios de tipo "dias de libre disposición" para lo que utilizarán plantillas.
 * Solo puede haber un calendario de tipo "dias de libre disposición" activo a la vez.
